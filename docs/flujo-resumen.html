Guía para perfiles no técnicos

¿Cómo funciona el proyecto CHEC UITI_VANO?

Una explicación sin nombres de archivos ni comandos, pensada para quien quiere entender el "qué" y el "por qué", no el "cómo" técnico. Para el detalle completo, ver flujo-detallado.md.

¿Para qué sirve esto?

CHEC opera líneas eléctricas de distribución divididas en circuitos. Cada circuito tiene muchos tramos pequeños llamados vanos. Cuando algo falla, el sistema registra ese evento y lo asocia a un indicador llamado UITI_VANO, que mide qué tan grave fue el impacto de esa falla para los usuarios conectados.

Este proyecto toma ese historial de eventos y responde dos preguntas: qué tan crítico es cada circuito comparado con los demás, y qué variables influyen más en que una falla sea grave. Dos caminos, con el mismo punto de partida, llegan a destinos distintos.

Camino 1 — Reporte por circuito Una página web local, redactada con ayuda de inteligencia artificial, lista para leer en cualquier navegador.
Camino 2 — Panel visual en la nube Un dashboard interactivo en Databricks, con mapas y gráficos filtrables.

Camino 1 — El reporte de un circuito

Se puede pedir para un circuito, un grupo de circuitos, o como informe gerencial resumido.

1
Selección de datos

Se eligen el circuito y el periodo de tiempo a analizar.

2
Detección automática

Un sistema (no una persona) revisa los datos y encuentra los puntos donde el impacto fue más alto.

3
Tres asistentes de IA en paralelo

Uno explica qué pasó; otro explica qué variables pesaron más en un modelo estadístico entrenado antes; un tercero prueba "qué pasaría si" cambiara una variable.

4
Comparación contra expertos

Un cuarto asistente compara ese análisis contra documentos técnicos de expertos humanos, señalando coincidencias o diferencias.

5
Reporte web

Todo se arma en una página HTML lista para abrir en el navegador, sin instalar nada.

Camino 2 — El panel visual en Databricks

Una copia independiente del análisis, publicada como dashboard interactivo en la nube.

1
Copia de los datos

El mismo historial de eventos se copia a un almacenamiento en la nube.

2
Copia del código real

No una versión reescrita a mano — exactamente el mismo código que corre localmente, para que los números siempre coincidan.

3
Reconstrucción de tablas

Se reconstruyen en la nube las mismas tablas de datos que respaldan el análisis.

4
Publicación del panel

El dashboard queda disponible con mapas y gráficos conectados a esas tablas.

Importante

El panel en la nube es una copia independiente. Si alguien cambia el análisis local, la copia en Databricks no se actualiza sola — hay que repetir el proceso de copiado para que refleje el cambio.

Lo que nunca se mezcla

La página web pública del proyecto (la que muestra los resultados de forma general) solo se genera y actualiza desde un computador local, nunca desde la nube. Esto evita que se acumulen archivos innecesarios o desactualizados en Databricks, y mantiene un único lugar responsable de esa publicación.

Glosario rápido

Circuito
Una línea eléctrica de distribución completa, con muchos tramos.
Vano
Un tramo pequeño dentro de un circuito, entre dos postes.
UITI_VANO
Un número que mide qué tan grave fue el impacto de una falla en un vano específico.
Criticidad
Qué tan grave es, en general, un circuito comparado con los demás — de "muy alto riesgo" a "riesgo bajo".
Agente de IA
Un asistente automático que lee datos y redacta una explicación en lenguaje natural, siguiendo reglas estrictas de validación.
Dashboard
Un panel visual interactivo en la nube, con gráficos y mapas que se pueden filtrar.
Databricks
La plataforma en la nube donde vive ese panel visual.
