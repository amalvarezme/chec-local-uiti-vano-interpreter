# LLM Workflow

The LLM layer explains a context package generated by deterministic Python code. It
does not select circuits, detect critical points, infer missing variables, or use
external evidence.

## Deterministic Analysis vs LLM Interpretation

Python code loads and validates the data, filters the selected window, builds the
daily `UITI_VANO` series, detects critical points, enriches them with attribution,
and saves a structured JSON context. The LLM receives that context and produces a
Spanish JSON interpretation.

## Skills

Markdown files under `llm/skills/` are playbooks for humans and coding agents. They
describe how to build context, interpret points, write grounded findings, and validate
LLM output. The notebook verifies that the required skill files exist and can be loaded.

## Prompt Templates

Templates live under `llm/prompts/`:

- `uiti_vano_explanation.system.md`
- `uiti_vano_explanation.user.md`
- `uiti_vano_explanation.output_schema.json`

`llm_contracts.py` assembles the final prompt with `{context_json}`,
`{output_schema_json}`, and `{prompt_version}`.

## JSON Schema Validation

LLM responses must be valid JSON and match the output schema. Additional guardrail
checks reject forbidden scope claims, dates outside the context, unknown critical point
IDs, and unavailable columns referenced as present.

## Local Evals

```bash
python llm/evals/run_llm_eval.py
```

The evals require no API key. They render prompts from synthetic contexts and validate
sample outputs that should pass or fail.

## Acceptable Claims

- "La evidencia tabular muestra un pico de `UITI_VANO` el 2026-01-05."
- "El patron es compatible con una concentracion de impacto en pocos dias."
- "Dentro de las variables disponibles, la duracion y los usuarios ayudan a explicar la magnitud."

## Unacceptable Claims

- "Segun la normativa..."
- "La bitacora evidencia..."
- "El modelo predice..."
- "La simulacion muestra..."
